ЦПП.

1.1. Связь с контроллером ЦПП: 
	UART0, 38400 бит/с, 8 бит данных, 1 стоп-бит, без контроля четности.
	
1.2. Протокол:
	Два байта данных, второй равен первому.
	Передавать данные в ЦПП необходимо с периодом 2мс.
	На каждую переданную посылку в ЦПП, обратно будет отправлена посылка с 
	текущим состоянием ЦПП (номером команды, состоянием входа ТМ или ошибкой).
	В случае наличия ошибки в работе ЦПП команды на передачу/прием по ЦПП будут
	блокированы.
	
	Назначение бит при передаче = abcccccc:
	a=0 , передается команда 
		b - логический уровень устанавливаемый на выход ТМ. 
		cccccc - номер команды на передачу [0..32]
	a=1 , передаются настройки 
		b=1, cccccc=1 - останов ЦПП 
		b=1, cccccc=2 - работа в качестве Приемника КЕДР 
		b=1, cccccc=3 - работа в качестве Передатчика КЕДР
		
	Назначение бит при приеме = abcccccc:
	a=0 , принимается команда. 
		b - логический уровень на входе ТМ. 
		cccccc - номер команды принятый с ЦПП [0..32].
	a=1 , принимается ошибка. 
		b - логический уровень на входе ТМ. 
		cccccc=1 - ЦПП не запущен. 
		cccccc=2 - ошибки в работе ЦПП для режима Приемник КЕДР. 
		cccccc=3 - ошибки в работе ЦПП для режима Передатчик КЕДР.
		
1.3. Параметры пользователя:
1.3.1 "Режим ЦС": 
	"Выкл" = 0, ЦС выключен; 
	"Приемник КЕДР" = 1, ЦС работает аналогично приемнику КЕДР;
	"Передатчик КЕДР" = 2, ЦС работает аналогично передатчику КЕДР.
1.3.2 "Команды трансл. ЦС": 
	"Выкл" = 0,	трансляция команды запрещена
	"Вкл" = 1, трансляция команды разрешена.
	Данный параметр настраивается для каждой команды аппарата в отдельности.
	В зависимости от текущего режима, это будет разрешение команды принимаемых 
	по ЦС или передаваемых по ЦС.
1.3.3 "Переназн. ком. ЦС":
	[1..32]. Параметр используемый только в случае работы ЦС в качестве 
	"Приемник КЕДР". Позволяет изменить номер команды передаваемый в ЦС при
	принятии команды ПА.
	Данный параметр настраивается для каждой команды аппарата в отдельности.

2. Общее описание работы.
При включении необходимо установить нужный режим работы ЦПП. Если в рабочем
цикле вернется ошибка "ЦПП не запущен", установить режим заново и сделать запись
в журнал.

Работа ЦПП в режиме "Приемник КЕДР".
В данном режиме, команды принимаемые по ЦПП будут проигнорированы и команда по 
приему всегда будет 0. 
При наличии команды с удаленного аппарата, необходимо проверить ее разрешение
на передачу по ЦПП и учесть переназначение команды ЦС.
Далее передавать ее в ЦПП пока реле команды будет замкнуто (?! или пока
не исчезнет сигнал в ВЧ канале).
Сделать запись в журнал.
При налчии ошибки связи ЦПП, формируется "предупреждение ЦС". Команды по ЦС при
этом должны передаваться.

Работа ЦПП в режиме "Передатчик КЕДР".
В данном режиме, команды передаваемые в ЦПП будут проигнорированы, команда 
всегда будет 0. 
При наличии команды принятой по ЦПП, необходимо проверить ее разрешение
на передачу. 
При наличии ошибки связи ЦПП, формируется "авария ЦС". При воостановлении связи - 
авария сбрасывается сама.
Сделать запись в журнал. 


////
При отсутствии связи по ЦПП Приемник КЕДР уходит в неисправность и команды не
принимает.

При отсутствии связи по ЦПП Передатчик уходит в предупреждение.

При принятии команды, Приемник КЕДР постоянно шлет ее номер по ЦПП.

При пропадании команды, на несколько мс Передатчик КЕДР снова шлет команду.
